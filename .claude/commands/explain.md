요구사항의 구현 방향을 사용자와 대화로 결정합니다.

사용자 입력: $ARGUMENTS

## 대상 선택

1. $ARGUMENTS에 REQ-ID (예: REQ-001)가 있으면 해당 요구사항을 선택한다
2. $ARGUMENTS가 비어있으면 wiki/requirements/README.md에서 첫 번째 [ ] 항목을 자동 선택한다
3. 미완료 항목이 없으면: "/add-req 커맨드로 요구사항을 먼저 추가하세요."를 출력하고 종료한다

## 실행 절차

1. 선택된 REQ의 상세 파일(wiki/requirements/REQ-NNN.md)을 읽는다
2. 프로젝트 코드베이스를 탐색하여 현재 구조와 관련 코드를 파악한다
3. 다음 항목에 대해 **구현 방향을 제안**한다:
   - 아키텍처/설계 방향
   - 변경 대상 파일 및 모듈
   - UI 변경 사항 (해당 시)
   - 데이터 모델 변경 (해당 시)
   - 기타 주요 결정 사항
4. 각 제안에 대해 사용자의 피드백을 요청한다
5. 사용자와 대화를 통해 방향을 확정한다

## 중요 규칙

- **일방적으로 결정하지 않는다**: 반드시 제안 → 사용자 피드백 → 확정 흐름을 따른다
- **한 번에 모든 것을 물어보지 않는다**: 핵심 사항부터 순서대로 제안하고 대화한다
- **사용자가 "그냥 알아서 해" 또는 명시적으로 위임하면**: 해당 사항을 "미정 (클로드 자율 결정)"으로 기록한다
- **모든 사항이 열려있어도 괜찮다**: /teams 실행 시 클로드가 자율적으로 판단한다

## 결과 기록

대화가 완료되면 REQ 파일(wiki/requirements/REQ-NNN.md)에 다음 섹션을 추가(append)한다:

```markdown
## 구현 방향 (explain)

> Date: {YYYY-MM-DD}

### 확정 사항
- {사용자가 결정한 사항 1}
- {사용자가 결정한 사항 2}

### 미정 사항 (teams 실행 시 자율 결정)
- {열려있는 사항 1}
- {열려있는 사항 2}
```

## 완료 안내

- 기록 완료 후 "/teams 커맨드로 구현을 시작할 수 있습니다."를 안내한다
- wiki/ 경로는 반드시 현재 작업 디렉토리(cwd) 기준으로 참조한다
